Welcome to Superset Documentation

About the Dashboard

The Integrated DHIS2-EMR Dashboard for Maternal and Child Health (MCH) is a unified platform that consolidates key health data from the DHIS2 (District Health Information System) and EMR (Electronic Medical Records) systems. It is designed to provide real-time, actionable insights for decision-makers across Ethiopia’s health system, particularly focusing on reproductive, maternal, newborn, and child health services.

Fast & Lightweight

Superset is designed to be easy to use while maintaining the ability to scale to large organizations.

Rich Visualizations

Create interactive dashboards with a wide variety of visualization options.

Multiple Databases

Connect to virtually any SQL-speaking datasource, also by uploading external files like Excel,Csv,and Connect to Google Sheet is possible .

Introduction to Superset

Overview

The Integrated DHIS2-EMR Dashboard, built using Apache Superset, visualizes MCH data from DHIS2, EMR, and uploaded files—helping health staff make informed decisions through interactive dashboards.

Key Concepts

  • Datasources: Connects to DHIS2, EMR, and uploaded CSV/Excel files for real-time or periodic MCH data.
  • Visualizations: Create charts and graphs to represent your data
  • Dashboards: Combine multiple visualizations into interactive dashboards
  • SQL Lab: A powerful SQL editor for advanced data exploration

Architecture

Superset is built on a modern tech stack:

Frontend: React + Redux
Backend: Python (Flask)
Database: SQLAlchemy (supports multiple databases)
Visualization: Apache ECharts, D3.js

PyPI Installation

Installing Superset via PyPI

Follow these steps to install Superset using the Python Package Index (PyPI).

Step 1: System Preparation

Update your system and install required dependencies:

# Update system packages sudo apt update -y && sudo apt upgrade -y # Install essential dependencies sudo apt-get install -y build-essential libssl-dev libffi-dev \ python3-dev python3-pip libsasl2-dev libldap2-dev \ default-libmysqlclient-dev python3.10-venv

Step 2: Create Application Directory

Set up a dedicated directory for Superset:

# Create application directory sudo mkdir /app sudo chown $USER /app cd /app # Create Superset environment mkdir superset cd superset python3 -m venv superset_env source superset_env/bin/activate # Update pip pip install --upgrade setuptools pip

Step 3: Install Superset

Install required Python packages and Superset:

# Install required dependencies pip install pillow # Install Apache Superset pip install apache-superset

Step 4: Configuration Setup

Create a configuration file and set environment variables:

# Create configuration file touch superset_config.py # Set configuration path export SUPERSET_CONFIG_PATH=/app/superset/superset_config.py

Edit superset_config.py with the following content:

# Superset configuration ROW_LIMIT = 5000 # Flask App Builder configuration SECRET_KEY = 'YOUR_OWN_RANDOM_GENERATED_SECRET_KEY' # Database configuration SQLALCHEMY_DATABASE_URI = 'sqlite:////app/superset/superset.db?check_same_thread=false' # Security settings TALISMAN_ENABLED = False WTF_CSRF_ENABLED = False # Mapbox API key (optional) MAPBOX_API_KEY = ''

Generate a secure secret key with this command:

openssl rand -base64 42

Step 5: Initialize Database

Set up the metadata database and create an admin user:

# Set Flask application export FLASK_APP=superset # Upgrade database superset db upgrade # Create admin user superset fab create-admin # Initialize Superset superset init

Step 6: Running Superset

Create a startup script to run Superset with Gunicorn:

# Create run script nano run_superset.sh # Script content: #!/bin/bash export SUPERSET_CONFIG_PATH=/app/superset/superset_config.py source /app/superset/superset_env/bin/activate gunicorn -w 10 -k gevent --timeout 120 -b 0.0.0.0:8088 \ --limit-request-line 0 --limit-request-field_size 0 \ "superset.app:create_app()" # Make script executable chmod +x run_superset.sh # Start Superset ./run_superset.sh

Step 7: Create System Service

Set up a systemd service for automatic startup:

# Create service file sudo nano /etc/systemd/system/superset.service # Service content: [Unit] Description=Apache Superset Webserver After=network.target [Service] Environment=SUPERSET_HOME=/app/superset Environment=PYTHONPATH=/app/superset WorkingDirectory=/app/superset ExecStart=/app/superset/run_superset.sh [Install] WantedBy=multi-user.target # Enable and start service sudo systemctl daemon-reload sudo systemctl enable superset.service sudo systemctl start superset.service

Verification

After completing these steps, your Superset instance should be running:

  • Access the web interface at http://your-server-ip:8088
  • Log in with the admin credentials you created
  • Verify the service status with: sudo systemctl status superset.service

Manual Installation

Manual Installation Guide

Follow these steps to install Superset manually:

Prerequisites

  • Python 3.7+
  • pip package manager
  • Virtual environment (recommended)

Installation Steps

1. Create and activate a virtual environment:

python3 -m venv venv
source venv/bin/activate

2. Install Superset:

pip install apache-superset

3. Initialize the database:

superset db upgrade

4. Create an admin user:

superset fab create-admin

5. Initialize Superset:

superset init

6. Start the development server:

superset run -p 8088 --with-threads --reload --debugger

Changing Superset Port

Modifying the Service Port

By default, Superset runs on port 8088. Follow these steps to change the port to your preferred value.

Important Considerations

  • Ensure the new port is not used by other services
  • Update firewall rules if needed
  • Restart the Superset service after making changes

Step 1: Locate the Startup Script

Navigate to your Superset installation directory:

cd /app/superset

Step 2: Edit the Startup Script

Open the run_superset.sh file in a text editor:

nano run_superset.sh

Current Script Content

#!/bin/bash
export SUPERSET_CONFIG_PATH=/app/superset/superset_config.py
. /app/superset/superset_env/bin/activate
gunicorn \
    -w 10 \
    -k gevent \
    --timeout 120 \
    -b 0.0.0.0:8088 \
    --limit-request-line 0 \
    --limit-request-field_size 0 \
    --statsd-host localhost:8125 \
    "superset.app:create_app()"

Step 3: Modify the Port Number

Find the line containing -b 0.0.0.0:8088 and change the port number to your desired value:

Before Modification

-b 0.0.0.0:8088 \

After Modification

-b 0.0.0.0:8888 \

For example, to change to port 8888:

# Change this line: -b 0.0.0.0:8088 \ # To: -b 0.0.0.0:8888 \

Step 4: Save and Exit

Save your changes and exit the editor:

# For nano editor: Press Ctrl+O to save, then Enter to confirm Press Ctrl+X to exit

Step 5: Restart the Service

Apply the changes by restarting the Superset service:

sudo systemctl restart superset.service

Step 6: Verify the Change

Check if Superset is running on the new port:

# Check service status sudo systemctl status superset.service # Check port usage sudo netstat -tuln | grep 8888

Verification

After completing these steps:

  • Access Superset at http://your-server-ip:8888
  • Confirm the service is listening on the new port:
    sudo lsof -i :8888

Superset Branding Customization

Customizing Your Superset Instance

Customize the appearance of Superset to match your organization's branding. All configurations are made in the superset_config.py file.

Important Notes

  • Place custom assets in /app/superset/static/assets/images/custom/
  • Restart Superset after making configuration changes
  • Clear browser cache to see immediate changes
  • Use PNG format for best results with transparent backgrounds

Setting Application Name

Customize the name displayed in the browser tab and throughout the application:

# Add to superset_config.py APP_NAME = "My Analytics Platform"

Before Customization

Apache Superset

After Customization

My Analytics Platform

Customizing the Logo

Replace the default Superset logo with your organization's logo:

# Add to superset_config.py # Path to your custom logo APP_ICON = "/static/assets/images/custom/logo.png" # Target path when clicking the logo LOGO_TARGET_PATH = '/' # Takes user to the welcome page # Tooltip when hovering over the logo LOGO_TOOLTIP = "Data Insights Portal" # Text to display next to the logo LOGO_RIGHT_TEXT = "Analytics Department"

Configuration Options

  • APP_ICON: Path to your logo file
  • LOGO_TARGET_PATH: URL path when logo is clicked
  • LOGO_TOOLTIP: Hover tooltip text
  • LOGO_RIGHT_TEXT: Additional branding text

Logo Requirements

  • Optimal size: 180×40 pixels
  • PNG format with transparency
  • Place in static assets folder
  • Ensure proper file permissions

Setting Favicon

Customize the browser tab icon (favicon) for your Superset instance:

# Add to superset_config.py FAVICONS = [ { "href": "/static/assets/images/custom/favicon.png" } ]

Favicon Specifications

  • Optimal size: 32×32 pixels
  • Multiple sizes for different devices
  • ICO format also supported
  • Place in static assets folder

Complete Branding Example

Here's a complete configuration example with all branding options:

# Complete branding configuration APP_NAME = "Enterprise Analytics" APP_ICON = "/static/assets/images/custom/enterprise-logo.png" LOGO_TARGET_PATH = '/' LOGO_TOOLTIP = "Enterprise Analytics Portal" LOGO_RIGHT_TEXT = "Data Insights Team" FAVICONS = [ { "href": "/static/assets/images/custom/favicon-32x32.png" } ]

Implementation Steps

  1. Create directory for custom assets:
    mkdir -p /app/superset/static/assets/images/custom
  2. Upload your logo and favicon to this directory
  3. Add branding configuration to superset_config.py
  4. Restart the Superset service:
    sudo systemctl restart superset.service
  5. Clear browser cache and refresh the page

Troubleshooting

  • Images not appearing:
    • Verify file paths are correct
    • Confirm files are in static assets directory
  • Changes not visible:
    • Restart Superset service
    • Clear browser cache (Ctrl+F5 or Cmd+Shift+R)
  • Poor image quality:
    • Use high-resolution source images
    • Ensure correct dimensions for each image type
